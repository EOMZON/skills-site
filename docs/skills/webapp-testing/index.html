Webapp Testing

在真实浏览器里验证本地 Web 应用，而不是只看源码或猜 UI 状态。

Use When

  • 需要检查交互流程、按钮行为、表单提交或页面跳转
  • 需要抓截图、日志或重现问题路径
  • 需要对上线前页面做最后一轮浏览器验收

Input

  • app_url_or_path：本地地址、页面入口或启动方式
  • verification_goal：关键流程、问题现象或验收标准

Returns

  • 浏览器内的验证结果
  • 截图、日志和问题证据
  • 可复跑的 Playwright 操作步骤

Notes

  • 公开版保留 Playwright 工作方式与黑盒脚本约定
  • 本地登录态、私有测试环境和个人浏览器状态不包含在公开版内

Invoke

$webapp-testing

Inputs

app_url_or_path 待验证的本地地址、页面入口或运行方式 (required)
verification_goal 要检查的关键流程、问题现象或验收标准

Returns

浏览器内的验证结论
截图、日志或复现证据
可复跑的自动化步骤

Use When

需要验证本地网页或前端交互
需要复现 bug、截图取证或查看浏览器日志
需要在真实浏览器里走一遍关键流程

Avoid When

只是阅读静态源码而不需要运行页面
不是 Web 界面相关问题
