Vercel Ship

把 Git 提交、GitHub 推送、Vercel 部署、Analytics 与自定义域名串成一条可重复执行的 ship 流水线。

GitHub First

完整公开说明放在 GitHub，不放在站内长期镜像。

这个页面只保留 discovery 和 contract。真正的公开说明、后续更新和 star 入口都应回到 GitHub source。

Open SKILL.md Open Folder Open Repo

Invoke

$vercel-ship

Inputs

project_path 待发布项目路径 (required)
shipping_overrides preview/prod、push、域名或 analytics 等覆盖项

Returns

已提交并推送的 Git 状态
Vercel 部署结果与线上地址
域名、Analytics 与后续验收信息

Use When

想把本地项目一口气 ship 到 GitHub 与 Vercel
想把 preview/prod、analytics 与域名处理成统一入口
需要一个稳定的发布流程，而不是手工拼几段命令

Avoid When

项目还没准备好发布
缺少 GitHub 或 Vercel 的鉴权环境
