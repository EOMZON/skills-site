Doc Co-Authoring

通过分阶段 workflow 把“信息很散、但必须写清楚”的文档任务推进成稿。

Use When

  • 需要写提案、Spec、PRD、决策文档或较长的说明文
  • 需要先把上下文讲清，再逐段打磨
  • 希望在交付前用读者视角检查理解成本

Input

  • doc_goal：文档类型、目标和主要读者
  • source_context：背景、模板、素材、已有草稿、会议记录

Returns

  • 文档骨架与章节顺序
  • 每一段的共创与修订建议
  • 发布前的缺口检查和读者测试提示

Notes

  • 公开版保留 workflow 和调用契约
  • 私有连接器、内部文档系统和团队上下文不在公开版内

Invoke

$doc-coauthoring

Inputs

doc_goal 文档类型、目标和主要读者 (required)
source_context 背景信息、模板、已有草稿、会议记录或相关链接

Returns

文档结构与章节 scaffold
逐段共创与修订建议
读者视角测试与缺口清单

Use When

要写结构化文档，但上下文还很散
希望边梳理边成稿，而不是一次性吐整篇
想在发出去前做一次读者视角检查

Avoid When

只需要润色一小段文本
还没有明确的文档目标或受众
