Scenario-First Skill Registry

先按场景进入，再选 skill。

首页先回答能做什么、从哪里起手、会拿到什么产出。公开层只保留可复用调用契约；需要收口运行细节的 workflow 用 sanitized entry 暴露，公开展示与机器入口都直接回到 repo-backed 的 source links。

Executable systems不是 prompt 片段。很多 skill 会直接落到报告、页面、脚本、部署或可复查产物。
Scenario-first先按要完成的事情分组，再在 scene 里看可调用的 skill，而不是先按工具名找。
Private source -> exposed manifests私有作者源先沉淀，再导出 repo-backed 的 exposed manifest；涉及状态和敏感运行细节的 workflow 以 sanitized 方式列出。
Agent-readable人看页面，AI 读 JSON 和 llms.txt。每个 skill 都有稳定的 machine entry point。
